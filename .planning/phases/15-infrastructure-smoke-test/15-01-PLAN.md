    **Create `scripts/` directory** (does not exist yet).

    **Create `scripts/smoke_gemini.py`** — a standalone script (not Django-dependent) that:

    1. Reads `GEMINI_API_KEY` from environment via `os.environ["GEMINI_API_KEY"]` (fail-fast)
    2. Accepts an optional CLI argument for the webm file path; defaults to generating a test file
    3. If no audio file is provided or the default path does not exist, generate a 3-second silent webm using ffmpeg:
       `ffmpeg -f lavfi -i anullsrc=r=48000:cl=stereo -t 3 -c:a libopus scripts/test_silence.webm -y`
       This avoids requiring the developer to source a test file.
    4. Creates a `genai.Client(api_key=api_key)`
    5. Uploads the file via `client.files.upload(file=audio_path, config=types.UploadFileConfig(mime_type="audio/webm"))`
    6. Prints the upload result: `uri`, `state`
    7. Adds a conservative `time.sleep(2)` after upload (handles PROCESSING state for small files — per research open question 1)
    8. Calls `client.models.generate_content(model="gemini-2.5-pro", contents=[uploaded_file, "Briefly describe what you hear in this audio."])`
    9. Checks `response.candidates[0].finish_reason` — if it indicates SAFETY block, prints warning but still considers smoke test informational (safety blocks are Phase 16 concern)
    10. Prints `response.text`
    11. Prints `SMOKE TEST PASSED` on success

    **Exact imports:**
    ```python
    import os
    import sys
    import time
    import subprocess
    from pathlib import Path
    from google import genai
    from google.genai import types
    ```

    **Do NOT:**
    - Import Django settings (this script is standalone)
    - Use `google.generativeai` (wrong SDK — per STATE.md decision, package is `google-genai`, import is `from google import genai`)
    - Use a preview model variant (use `gemini-2.5-pro` exactly)
    - Add complex retry/polling logic for file state (simple sleep is sufficient for smoke test)
  </action>
