  <action>
Create session/services/assessment.py with the following structure:

1. Module-level constants:
   - `CRITERION_MAP` dict mapping tool_use response keys to SpeakingCriterion enum values:
     - "fluency_and_coherence" -> SpeakingCriterion.FC (1)
     - "grammatical_range_and_accuracy" -> SpeakingCriterion.GRA (2)
     - "lexical_resource" -> SpeakingCriterion.LR (3)
     - "pronunciation" -> SpeakingCriterion.PR (4)
   - `REQUIRED_KEYS = set(CRITERION_MAP.keys())`
   - `SYSTEM_PROMPT` -- a system prompt instructing Claude as an expert IELTS Speaking examiner. Include concise band descriptor summaries for each criterion (FC, GRA, LR, PR) covering band ranges 1-3, 4-5, 6-7, 8-9. Instruct to produce 3-4 sentences of actionable, specific feedback per criterion (per AIAS-03). Instruct to assess based on the provided transcript and session questions.
   - `TOOL_DEFINITION` dict defining tool "submit_ielts_assessment" with input_schema containing 4 required criterion objects, each with "band" (integer, min 1, max 9) and "feedback" (string). Use the exact schema structure from 12-RESEARCH.md Pattern 1.

2. `assess_session(job) -> list[dict]` function:
   - Deferred import of `anthropic` inside function body (try/except ImportError -> RuntimeError). Follow the exact pattern from transcription.py.
   - Deferred import of `django.conf.settings` and `session.models.SessionQuestion`.
   - Build question context (per AIAS-04): query SessionQuestion objects for the job's session, using `select_related("question", "session_part")` and `order_by("session_part__part", "order")`. Format as "[Part N] question text" lines. Default to "(no questions recorded)" if empty.
   - Build user_message combining questions_text and job.transcript (default "(no transcript)" if empty).
   - Create `anthropic.Anthropic(api_key=getattr(settings, "ANTHROPIC_API_KEY", None))` client.
   - Call `client.messages.create()` with:
     - model="claude-sonnet-4-20250514" (locked decision)
     - max_tokens=1024
     - system=SYSTEM_PROMPT
     - tools=[TOOL_DEFINITION]
     - tool_choice={"type": "tool", "name": "submit_ielts_assessment"} (locked decision: forced tool_use)
     - messages=[{"role": "user", "content": user_message}]
   - Extract tool_use block from response.content. If None, raise RuntimeError with stop_reason.
   - Validate all 4 REQUIRED_KEYS present in tool_block.input. If missing, raise RuntimeError (locked decision: all-or-nothing).
   - Validate each band is int in range 1-9. If invalid, raise RuntimeError.
   - Return list of 4 dicts: [{"criterion": int, "band": int, "feedback": str}, ...]
   - Add logging: INFO on API call start and completion with band values.

Do NOT import anthropic at module top level (project convention).
Do NOT catch anthropic exceptions inside assess_session -- let them propagate to the task's except block.
Do NOT use temperature=0 (unnecessary with forced tool_use per research).
  </action>
