    ```python
    def _validate_slot_available(examiner_id, slot_id, requested_date):
        """Validate that the given slot is available on the requested date using compute_available_slots."""
        from .services.availability import compute_available_slots
        week_data = compute_available_slots(examiner_id, requested_date)
        target_date_str = requested_date.isoformat()
        day_data = next((d for d in week_data if d["date"] == target_date_str), None)
        if not day_data:
            raise ValidationError("Requested date not found in examiner's schedule.")
        slot_data = next((s for s in day_data["slots"] if s["slot_id"] == slot_id), None)
        if not slot_data:
            raise ValidationError("This time slot does not exist in the examiner's schedule.")
        if slot_data["status"] != "available":
            raise ValidationError(f"This slot is {slot_data['status']} and cannot be booked.")
    ```

    - POST: Only examiners can accept (check _is_examiner, return 403 otherwise).
      1. Inside `with transaction.atomic():`
         a) `req = SessionRequest.objects.select_for_update().get(pk=pk, examiner=request.user)` — catch DoesNotExist, return 404
         b) `req.accept()` — raises ValidationError if not PENDING (propagates through DRF per v1.1 decision)
         c) Build scheduled_at: `datetime.combine(req.requested_date, req.availability_slot.start_time, tzinfo=dt_timezone.utc)`
         d) `session = IELTSMockSession.objects.create(examiner=req.examiner, candidate=req.candidate, scheduled_at=scheduled_at)` — status defaults to SCHEDULED
         e) `req.session = session`
         f) `req.save(update_fields=["status", "session", "updated_at"])`
      2. AFTER the atomic block (per v1.1 broadcast discipline):
         `_broadcast(session.pk, "session_request.accepted", {"session_request_id": req.pk, "session_id": session.pk})`
      3. Return SessionRequestSerializer(req).data

    **RejectRequestView (APIView):**

      4. `req.save(update_fields=["status", "rejection_comment", "updated_at"])`
