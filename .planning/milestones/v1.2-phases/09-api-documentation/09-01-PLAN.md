**docs/api/profiles.md** — Title: "# Profiles". Document these 5 endpoints:

1. `GET /api/profiles/examiner/me/` — Examiner only. Returns own profile with phone visible (detail view).
   Response 200 shape: `{ "id": 1, "user": { "id": 1, "username": "...", "email": "...", "first_name": "...", "last_name": "..." }, "bio": "...", "full_legal_name": "...", "phone": "+998...", "profile_picture": "https://...", "is_verified": true, "completed_session_count": 5, "credential": { "id": 1, "listening_score": 8.5, "reading_score": 8.0, "writing_score": 7.5, "speaking_score": 9.0, "certificate_url": "https://..." } }`
   Errors: `404` — `"Examiner profile not found."` (non-examiner user)

2. `PATCH /api/profiles/examiner/me/` — Examiner only. Partial update. Writable fields: bio, full_legal_name, phone, profile_picture. Read-only: is_verified, completed_session_count.
   Request example: `{ "bio": "IELTS examiner with 10 years experience" }`
   Response 200: same shape as GET.
   Errors: `404` — `"Examiner profile not found."`, `400` — Validation errors

3. `GET /api/profiles/examiner/<id>/` — Any authenticated user. Public view (phone hidden).
   Response 200: same as detail but WITHOUT `phone` field.
   Errors: `404` — `"Examiner profile not found."`

4. `GET /api/profiles/examiner/me/credential/` — Examiner only.
   Response 200: `{ "id": 1, "listening_score": 8.5, "reading_score": 8.0, "writing_score": 7.5, "speaking_score": 9.0, "certificate_url": "https://..." }`
   Errors: `404` — `"Examiner profile not found."` | `"No credential found."`

5. `PUT /api/profiles/examiner/me/credential/` — Examiner only. Create or update (idempotent).
   Request: `{ "listening_score": 8.5, "reading_score": 8.0, "writing_score": 7.5, "speaking_score": 9.0, "certificate_url": "https://..." }`
   Response 200: credential object.
   Errors: `404` — `"Examiner profile not found."`, `400` — Validation errors

6. `GET /api/profiles/candidate/me/` — Candidate only. Returns own profile with score history.
   Response 200: `{ "id": 1, "user": { ... }, "profile_picture": "https://...", "target_speaking_score": 7.0, "current_speaking_score": 6.5, "score_history": [{ "id": 1, "session_id": 5, "overall_band": 6.5, "created_at": "..." }] }`
   Errors: `404` — `"Candidate profile not found."`

7. `PATCH /api/profiles/candidate/me/` — Candidate only. Partial update. Writable: profile_picture, target_speaking_score, current_speaking_score.
   Validation: scores must be 1.0-9.0 in 0.5 increments.
   Errors: `404` — `"Candidate profile not found."`, `400` — `"Score must be between 1.0 and 9.0."` | `"Score must be a multiple of 0.5."`

8. `GET /api/profiles/candidate/<id>/` — Any authenticated user. Public view (read-only, includes score_history).
   Errors: `404` — `"Candidate profile not found."`

**docs/api/availability.md** — Title: "# Availability". Document these 6 endpoints:

1. `GET /api/scheduling/availability/` — Examiner only. List own recurring slots.
   Response 200: `[{ "id": 1, "day_of_week": 0, "start_time": "09:00:00", "created_at": "..." }]`
   Note: day_of_week: 0=Monday through 6=Sunday.
   Errors: `403` — `"Only examiners can view availability slots."`

2. `POST /api/scheduling/availability/` — Examiner only. Create slot.
   Request: `{ "day_of_week": 0, "start_time": "09:00" }`
   Validation: start_time must be on the hour, between 08:00 and 21:00.
   Response 201: slot object.
   Errors: `403` — `"Only examiners can create availability slots."`, `400` — `"start_time must be on the hour (e.g., 08:00, 14:00)."` | `"start_time must be between 08:00 and 21:00 inclusive."` | `"You already have an availability slot for this day and time."`

3. `PATCH /api/scheduling/availability/<id>/` — Examiner only. Update own slot.
   Errors: `403` — `"Only examiners can update availability slots."`, `404` — Not found

4. `DELETE /api/scheduling/availability/<id>/` — Examiner only. Response 204 (no body).
   Errors: `403` — `"Only examiners can delete availability slots."`, `404` — Not found

5. `GET /api/scheduling/blocked-dates/` — Examiner only. List own blocked dates.
   Response 200: `[{ "id": 1, "date": "2024-06-15", "reason": "Holiday", "created_at": "..." }]`
   Errors: `403` — `"Only examiners can view blocked dates."`

6. `POST /api/scheduling/blocked-dates/` — Examiner only. Create blocked date.
   Request: `{ "date": "2024-06-15", "reason": "Holiday" }`
   Response 201: blocked date object.
   Errors: `403` — `"Only examiners can create blocked dates."`

7. `DELETE /api/scheduling/blocked-dates/<id>/` — Examiner only. Response 204.
   Errors: `403` — `"Only examiners can delete blocked dates."`, `404` — Not found

8. `GET /api/scheduling/examiners/<id>/available-slots/` — Any authenticated user. Computed available slots for a week.
   Query params: `?week=2024-06-10` (required, ISO date), `?timezone=Asia/Tashkent` (optional, informational).
   Response 200: `[{ "date": "2024-06-10", "day_of_week": 0, "slots": [{ "slot_id": 1, "start_time": "09:00", "status": "available" }] }]`
   Slot status values: "available", "booked", "blocked".
   Errors: `400` — `"week query parameter is required."` | `"Invalid week format. Use YYYY-MM-DD."`, `404` — Examiner not found

9. `GET /api/scheduling/examiners/<id>/is-available/` — Any authenticated user. Real-time availability check.
   Response 200: `{ "is_available": true, "current_slot": { ... } }` or `{ "is_available": false, "current_slot": null }`
   Errors: `404` — Examiner not found

**docs/api/requests.md** — Title: "# Session Requests". Document these 4 endpoints:

1. `GET /api/scheduling/requests/` — Both roles. Examiner sees requests where they are examiner; candidate sees own requests.
   Query params: `?status=0` (0=PENDING, 1=ACCEPTED, 2=REJECTED, 3=CANCELLED).
   Response 200: `[{ "id": 1, "candidate": 2, "examiner": 1, "availability_slot": 5, "requested_date": "2024-06-12", "session": null, "comment": "...", "rejection_comment": null, "status": 0, "created_at": "...", "updated_at": "..." }]`

2. `POST /api/scheduling/requests/` — Candidate only. Submit request.
   Request: `{ "availability_slot": 5, "requested_date": "2024-06-12", "comment": "Looking forward to the session" }`
   Validation: requested_date weekday must match slot's day_of_week; date cannot be in the past; slot must be available (not booked/blocked).
   Response 201: request object.
   Errors: `403` — `"Only candidates can submit session requests."`, `400` — `"Requested date does not match the slot's day of week."` | `"Requested date cannot be in the past."` | `"This slot is booked and cannot be booked."` | `"You already have an active request for this slot and date."`

3. `POST /api/scheduling/requests/<id>/accept/` — Examiner only. Atomically creates linked IELTSMockSession.
   Response 200: request object with session field populated.
   Broadcasts WS event: `session_request.accepted`
   Errors: `403` — `"Only examiners can accept session requests."`, `404` — `"Not found."`, `400` — ValidationError if not PENDING

4. `POST /api/scheduling/requests/<id>/reject/` — Examiner only. Requires rejection_comment.
   Request: `{ "rejection_comment": "I am unavailable on that date." }`
   Response 200: request object with rejection_comment populated.
   Errors: `403` — `"Only examiners can reject session requests."`, `404` — Not found, `400` — rejection_comment required

5. `POST /api/scheduling/requests/<id>/cancel/` — Candidate or examiner. Must be PENDING or ACCEPTED.
   Response 200: request object with status=3 (CANCELLED).
   Errors: `403` — `"You are not a participant in this request."`, `404` — `"Not found."`, `400` — ValidationError if not PENDING/ACCEPTED
